# src 层说明

## 这一层做什么
`src/` 是仓库里真正的源代码根。这里定义“模组是什么”，而不是“模组被怎样发布”。如果只关心功能实现、运行时资源和源码布局，这一层是最核心的入口。

## 实现方式
项目采用 Gradle 默认约定，只使用了 `main` source set，没有 `test`、`androidTest` 之类并列结构。这说明仓库当前定位非常明确：一套生产代码，围绕 Mindustry 模组运行时展开。

## 与其他层级的关系
- 它向上受根层 `build.gradle` 管理。
- 它向下拆成 `java/` 和 `resources/`，分别承载逻辑与文案。
- `build/` 的绝大多数内容都由这里派生，所以这一层是整个构建图里的“源头层”。
