# resources 层说明

## 这一层做什么
这一层存放运行时资源，而不是 Java 代码。在本项目里，资源几乎全部集中在 bundle 文案上，用于设置界面与提示文本的多语言显示。

## 实现方式
当前只有 `bundles/`，这说明项目没有额外图片、音频、着色器或配置模板资源。换句话说，这一层的职责非常聚焦：提供本地化文本，并让 Java 代码通过键值而非硬编码字符串取文案。

## 与其他层级的关系
- 它由 `src/main` 管理，并在构建时复制到 `build/resources/main`。
- Java 代码中的 `@pss.category`、`setting.pss-enabled.name` 等键都依赖这里定义。
- 最终发布包中的 `bundles/` 目录就是这一层的直接镜像。
